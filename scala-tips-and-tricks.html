Concepts

Case class

Case class in Scala is simply a class with 2 additional properties:

  1. Case classes hold immutable data and are therefore very useful for modelling such use cases
  2. Case classes are very helpful for pattern-matching - they solve the problem avoiding the need to iterate-and-match. An example is as
    case class Dog(breed:String,name:String,age:Int)

for(dog<-List(leo,fluffy,candy)){
	dog match{
		case Dog("Dogue de Bordeaux","Leo",2)=>println("Hi, Leo")
		case Dog("Lesser Indian Spitz","Fluffy",8)=>println("Hi, Fluffy")
		case Dog(breed,name,age)=>println("Hi, "+name)
	}
}

Scala Testing Suite (scalatest)

Installation

scalatest needs to be added to the libraryDependencies in build.sbt as:

libraryDependencies += "org.scalatest" % "scalatest_2.13" % "3.0.8"
libraryDependencies += "org.scalatest" % "scalatest_2.13" % "3.0.8" % "test"

Getting started

Start by importing the FunSuite (functional suite) and other relevant libraries as:

import org.scalatest.FunSuite
import org.scalatest.BeforeAndAfter

Make sure you extend the new testing class with FunSuite and BeforeAndAfter

Heres' a simple test class:

import org.scalatest.FunSuite
import org.scalatest.BeforeAndAfter

class SimpleTests extends FunSuite with BeforeAndAfter {

  test("Creating an empty Parking Lot") {
    val emptyParkingLot: ParkingLot = new ParkingLot(0)
    assert(emptyParkingLot.findNearestEmptySlot() ==  -1)
  }
}

If some statements have to be compulsorily executed before the unit tests, use before:

before {
    testParkingLot = new ParkingLot(10)
  }

Using file-based Configurations in Scala

Using ConfigFactory in spark-shell

Assuming application.conf is available in the root directory, start the spark-shell as spark-shell --jars "config-1.2.1.jar" --files "application.conf"

import com.typesafe.config.ConfigFactory
import java.io.File
import org.apache.spark.SparkFiles

val config = ConfigFactory.parseFile(new File(SparkFiles.get("application.conf")))

Using ConfigFactory in Spark job

Usually application.conf is used as the configuration file, and it should be available in PATH.

import com.typesafe.config.ConfigFactory
val config = ConfigFactory.load()

Getting configuration parameters

config.getString("my.organization.project.name")

sbt - Scala's ultimate build tool

How to compile the Scala classes into a JAR?

sbt package is the command to compile the Scala classes in /src/main/scala-x.xx/ClassName.class. This should produce the JAR at /target/scala-x.xx/ClassName.jar.

What is a fat (Uber) JAR and how to build one?

Fat JAR packages ALL dependencies into a single JAR (which makes it highly versatile) and can run anywhere without any dependency installs. To create a fat JAR, w must use the command sbt assembly. Read more:

To create fat JAR in sbt, we need to add the following plugin information to assembly.sbt:

addSbtPlugin("com.eed3si9n" % "sbt-assembly" % "0.14.5")

Also, we need to take care of the deduplication problem by using a MergeStrategy. The following code needs to be appended to build.sbt:

val meta = """META.INF(.)*""".r
assemblyMergeStrategy in assembly := {
  case PathList("javax", "servlet", xs @ _*) => MergeStrategy.first
  case PathList(ps @ _*) if ps.last endsWith ".html" => MergeStrategy.first
  case n if n.startsWith("reference.conf") => MergeStrategy.concat
  case n if n.endsWith(".conf") => MergeStrategy.concat
  case meta(_) => MergeStrategy.discard
  case x => MergeStrategy.first
}

Where do you put in all the dependencies?

In build.sbt, the syntax is libraryDependencies += "com.paulgoldbaum" %% "scala-influxdb-client" % "0.6.1"

Using an external dependency using the REPL?

sbt console within the main directory where build.sbt is located.

How can you sbt assembly skipping the tests?

sbt 'set test in assembly := {}' assembly

Some Advanced Operations

DataFrame comparison

Often as a part of testing operations, we need to compare DataFrames, and we required features like non-sorted payloads, exact diff-highlighting etc. Following are some relevant libraries in this context:

MrPowers/spark-fast-tests

A fast Apache Spark testing helper library with beautifully formatted error messages! Works with scalatest and uTest. The assertSmallDatasetEquality method can be used to compare two Datasets (or two DataFrames). The assertSmallDatasetEquality method can also be used to compare Datasets. The colors in the error message make it easy to identify the rows that aren't equal.

https://github.com/MrPowers/spark-fast-tests

AbsaOSS/hermes

A E2E test tool for Enceladus. Also general dataset/dataframe comparison tool - AbsaOSS/hermes

https://github.com/AbsaOSS/hermes

Common sbt problems and their solutions

How to compile the Scala classes into a JAR?

sbt package is the command to compile the Scala classes in /src/main/scala-x.xx/ClassName.class. This should produce the JAR at /target/scala-x.xx/ClassName.jar.

What is a fat (Uber) JAR and how to build one?

Fat JAR packages ALL dependencies into a single JAR (which makes it highly versatile) and can run anywhere without any dependency installs. To create a fat JAR, w must use the command sbt assembly. Read more:

To create fat JAR in sbt, we need to add the following plugin information to assembly.sbt:

addSbtPlugin("com.eed3si9n" % "sbt-assembly" % "0.14.5")

Also, we need to take care of the deduplication problem by using a MergeStrategy. The following code needs to be appended to build.sbt:

val meta = """META.INF(.)*""".r
assemblyMergeStrategy in assembly := {
  case PathList("javax", "servlet", xs @ _*) => MergeStrategy.first
  case PathList(ps @ _*) if ps.last endsWith ".html" => MergeStrategy.first
  case n if n.startsWith("reference.conf") => MergeStrategy.concat
  case n if n.endsWith(".conf") => MergeStrategy.concat
  case meta(_) => MergeStrategy.discard
  case x => MergeStrategy.first
}

Where do you put in all the dependencies?

In build.sbt, the syntax is libraryDependencies += "com.paulgoldbaum" %% "scala-influxdb-client" % "0.6.1"

Using an external dependency using the REPL?

sbt console within the main directory where build.sbt is located.

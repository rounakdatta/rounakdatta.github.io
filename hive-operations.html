Using and Knowing Hive better

Posted on Mon, Jul 8, 2019 Hive

What's Hive and where to use it?

Hive is a SQL-based query engine working with SQL databases as well as data warehouses like AWS S3. Hive natively works with its MapReduce, however Tez is the modern, more-powerful query-execution engine behind Hive. Hive might be a popular data store choice for your Data Platform.

  • You have databases and tables like MySQL on Hive.
  • You have Hive Metastore which stores all the metadata for the particular table - schema, location (if external table), partition information, compression details, filetype format etc.
  • You can have data stored in external locations (e.g. S3) in columnar format (Parquet, ORC are the best columnar data formats).

Understand Hive Metastore

Hive Metastore

Spark SQL uses a Hive metastore to manage the metadata of persistent relational entities (e.g. databases, tables, columns, partitions) in a relational database (for fast access). A Hive metastore warehouse (aka spark-warehouse) is the directory where Spark SQL persists tables whereas a Hive metastore (aka metastore_db) is a relational database to manage the metadata of the persistent relational entities, e.g.

Useful Hive commands

Getting the list of all the tables

SHOW TABLES;

Know the command behind creating a table

SHOW CREATE TABLE <tableName>;

Show the partitions for a table

SHOW PARTITIONS <tableName>;

See the detailed description (schema + other info) of a table

DESCRIBE EXTENDED <tableName>;

Create a new external table

CREATE EXTERNAL TABLE <tableName> (<column> <dataType>) LOCATION '<externalLocation>';

Change the location of an external table

ALTER TABLE <tableName> SET LOCATION '<externalLocation>';

Repair the partitions of an external table

MSCK REPAIR TABLE <tableName>;

Insert Overwrite a table (means write data from another location into this table's current location)

INSERT OVERWRITE TABLE <destinationTableName> PARTITION(<partitionCol1>, <partitionCol2>, .. <partitionColn>) SELECT * FROM <originTableName> WHERE <condition> DISTRIBUTE BY <randomEnoughColumn>;

Seeing the external location for a particular partition

SHOW TABLE EXTENDED LIKE <tableName> PARTITION (<partitionCol1>=<partitionVal1>, <partitionCol2>=<partitionVal2>, ...);

What should I use as the Hive execution engine?

To use pure MapReduce, start Hive shell as hive --hiveconf hive.execution.engine=mr.

To use Tez, start Hive shell as hive --hiveconf tez.queue.name=<yarnQueueName>.

How to run a query saved in a file through Hive? In other words, how to automate Hive / Tez jobs through a Shell script?

nohup hive --hiveconf tez.queue.name=<queueName> -f query.sql &

How to use beeline, the better result parser for Hive?

Start the Hive shell as:

beeline -u jdbc:hive2://hiveuri -n hive

When you're using an authenticated connection,

beeline -u jdbc:hive2://$BEELINE_EP -n $BEELINE_USER -p $BEELINE_PWD

Hyperloglog | Enqueue Zero

HyperLogLog is an algorithm that can solve Count Distinct problem. A Count Distinct problem is something like getting number 5 for a data set like [1, 3, 2, 1, 5, 2, 4], for it has [1, 2, 3, 4, 5] 5 elements. HyperLogLog can provide estimated count on a very large data stream.

https://enqueuezero.com/hyperloglog.html

Approximate Aggregate Functions

count(distinct value) can often cause memory exhausted errors where input data and the cardinality of value are large. HyperLogLog is an efficient algorithm for approximating the number of distinct elements in a multiset. Hivemall implements HyperLogLog++ in approx_count_distinct. approx_count_distinct is less accurate than COUNT(DISTINCT expression), but performs better on huge input.

https://hivemall.incubator.apache.org/userguide/misc/approx.html

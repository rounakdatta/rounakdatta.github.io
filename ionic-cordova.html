Hands-on with Ionic Cordova

Posted on Fri, Aug 23, 2019 Android iOS Cross-Platform App

Getting started

Install ionic, cordova

npm install -g ionic cordova

Initialize a new project

ionic start <appName> --type=ionic-angular

Changing the root page

Go to app.component.ts and change the variable rootPage.

Creating a new page

ionic generate page <pageName>

When a new page is created, make sure you are adding the PageName to declarations: [] and entryComponents: [] in app.module.ts.

Compiling Ionic app

iOS

npm install
ionic cordova platform add ios
ionic cordova build ios

android

npm install
ionic cordova build # select android
ionic cordova run android --device --verbose

Debugging Errors

I am trying to debug the Android app on Mac, but getting through errors like XCode CLT and SDK. How to proceed?

Install XCode CLI tools as:

xcode-select --install

Next up, you might have to accept all the licenses for Android SDK.

~/Library/Android/sdk/tools/bin/sdkmanager --licensesioio

I made changes to the app source code on Git â†’ then git pulled â†’ re-compiled the iOS app using XCode, but the changes are not reflected. How to fix?

The one and only solution to this is to re-build the iOS app. Going to the root source code directory run:

ionic cordova build ios

Next, when you hit Build on XCode, the newer version of the app will come up.

There are problems installing certain npm packages. For example cordova-sqlite-storage. How to fix?

ionic cordova plugin rm cordova-sqlite-storage
npm install -g cordova-sqlite-storage@latest
ionic cordova plugin add cordova-sqlite-storage

Often, in the worst case, you might need to remove the iOS platform and re-add the platform. The commands for those are already given. Also refer https://github.com/xpbrew/cordova-sqlite-storage/issues/856#issuecomment-497298630.

There are signing issues while building the app on a real iOS device. How to fix?

  1. Go to the XCode Source Code Explorer on the left (where you can browse all the files).
  2. Clicking on the root directory takes you to the Build and Signing options for the same.
  3. There, under Debug Release, select a iOS Developer account. If the organization name is being identified as error, try adding an extra random character to it :p

There are certain permission issues on iOS, for example NSBluetooth. How to fix?

Find the {ProjectName}-Info.plist file on the root directory. To solve the permissions issues, you need to add each of the required permissions within the XML file with an explanation of why you require it.

<key>ABCPermission</key>
<value>This permission will allow the user to use Bluetooth</value>

The app crashes if certain BLE broadcasters are detected (typically LG TV). How to fix?

This is a known error - and I didn't fix this yet on my app. An exception handling could help for the time being.

While making HTTP requests to a remote server from the app, it gives CLEARTEXT_NOT_PERMITTED error for non-SSL server IPs. How to fix this?

Firstly, it is highly recommended that you set up your SSL (HTTPS) for your server using LetsEncrypt because that's the recommended standard. If you're looking for a immediate lookaround,

project-root/resources/android/xml/network_security_config.xml

<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
   <domain-config cleartextTrafficPermitted="true">
     <domain>ip-or-domain-name</domain>
   </domain-config>
</network-security-config>

project-root/config.xml

<platform name="android">
    <edit-config file="app/src/main/AndroidManifest.xml" mode="merge" target="/manifest/application" xmlns:android="http://schemas.android.com/apk/res/android">
        <application android:usesCleartextTraffic="true" />
        <application android:networkSecurityConfig="@xml/network_security_config" />
    </edit-config>
    ... other statements...

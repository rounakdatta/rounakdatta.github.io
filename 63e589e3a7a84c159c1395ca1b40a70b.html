Pushing metrics from Scala to InfluxDB

InfluxDB Grafana Scala Hotstar

Interacting with InfluxDB through curl commands

To check which all databases are there in InfluxDB [GET],

curl -G http://10.6.35.68:8086/query?pretty=true -u hotstar:hotstar789@ --data-urlencode "q=SHOW DATABASES"

To check the series present in a particular database [GET],

curl -G http://10.6.35.68:8086/query?pretty=true -u hotstar:hotstar789@ --data-urlencode "db=filecompaction" --data-urlencode "q=SHOW SERIES"

To create a new database [POST],

curl -i -POST http://10.6.35.68:8086/query?pretty=true -u hotstar:hotstar789@ --data-urlencode "q=CREATE DATABASE filecompaction"

To write a new point to a database,

curl -i -POST http://10.6.35.68:8086/write?db=filecompaction --data-binary 'filecompaction19072019,cd=2019-07-19,hr=04 originalsize=1234,compactedsize=432 1434055562000000000'

Get the points from a single measurement

curl -G http://10.6.35.68:8086/query?pretty=true -u hotstar:hotstar789@ --data-urlencode "db=hotstar" --data-urlencode "q=SELECT * FROM \"events-compacted\""

Interacting with InfluxDB from Scala

import com.paulgoldbaum.influxdbclient._
import scala.concurrent.ExecutionContext.Implicits.global

// connect to the influxdb daemon and then the required database
val influxdb = InfluxDB.connect("10.6.35.68", 8086, "hotstar", "hotstar789@")
val database = influxdb.selectDatabase("hotstar")

// create a new point (name, tags, fields)
var point = Point("events-compacted").addTag("event", "watched_video").addTag("cd", "2019-07-19").addTag("hr", hourNow).addField("originalSize", originalSize)

// push the point to influxdb
database.write(point)

FAQ

What is the difference between tag and field in InfluxDB?

Tags are indexed and can be used in quick search operations. Fields are not indexed and InfluxDB needs to move to the record offset and read the entire thing.

How to solve the error for netty.asyncHTTPClient?

libraryDependencies += "io.netty" % "netty-all" % "4.1.37.Final"

Creating locations

server {
	server_name 1.2.3.4
	absolute_redirect off;

	location /foobar {
		proxy_pass http://1.2.3.4/foobar;
		rewrite ^([^.]^[^/])$ $1/ permanent;
	}

	location /phpmyadmin {
		proxy_set_header Host $host:8080;
		proxy_pass http://1.2.3.4:8080/phpmyadmin;
}

The grand trouble of trailing slashes in URL

  1. Make sure while defining the API endpoints in your framework, define them to be containing slashes in the end
  2. Make sure the location definition in nginx config does not contain a trailing backslash
  3. Adding rewrite ^([^.]*[^/])$ $1/ permanent; to the location does the trick of redirecting all the un-slashed URLs to slashed ones

To proxy PHP-based apps (not Laravel or the similar), we would require a PHP socket to be running to serve these requests. An example is shown as:

	location ~* \.php(?:$|/) {
		include snippets/fastcgi-php.conf;
		fastcgi_param SCRIPT_FILENAME $request_filename;
		fastcgi_param modHeadersAvailable true; #Avoid sending the security headers twice
		fastcgi_pass unix:/run/php/php7.2-fpm.sock;
	}

Understanding nginx

What is a Reverse Proxy?

Think of Donald Trump and Kim Jong-un at a summit. Each know their languages, but not the either's. So, both need a scribe. If Kim Jong-un is the client (asking questions and making requests) and Trump the server (the host - or the person answering and giving responses), then the following are the entities in computer science language:

  1. Kim Jong-un - Client
  2. Kim Jong-un's scribe - Proxy
  3. Trump's scribe - Reverse Proxy
  4. Trump - Server

The presence of 1 and 4 is essential for the existence of a client-server architecture, however 2 and 3 are optional. Hope you understand Reverse Proxy now. Reverse Proxy always sits in front of the server (and sometimes protects it from overwhelming).

What is an upstream server?

Upstream servers are those servers which nginx can redirect the requests.

Hotstar Hadoop S3

Copying single file from local machine to remote machine (not Hadoop actually)

sudo scp -i <.pemFile> <locaLocationPath> <username>@<remoteMachineIP>:<remoteLocationPath>

Example command:

sudo scp -i keys/prod-dp-emr-nv.pem /Users/rounakd/IdeaProjects/jhousing/SmallFileCompaction/target/scala-2.11/smallfilecompaction_2.11-0.1.jar cloudbreak@10.10.241.22:/home/cloudbreak/rounak/smallfilecompaction_2.11-0.1.jar

Distributed copy from a location to another (locations can be HDFS/S3)

hadoop distcp -Dmapred.job.queue.name=<queueName> -Dfs.s3n.awsAccessKeyId=<accessKeyId> -Dfs.s3n.awsSecretAccessKey=<secretAccesskey> -m <numberOfMappers> s3a://<originS3Location> s3n://<destinationS3Location>

Example command:

hadoop distcp -Dmapred.job.queue.name=marketing -Dfs.s3n.awsAccessKeyId=xxx -Dfs.s3n.awsSecretAccessKey=yyy -m 200 s3a://hotstar-dp-datalake-processed-us-east-1-prod/events/watched_video/cd=2019-06-16 s3n://hotstar-dev-hive-warehouse-nv/events/indvpak_cp/cd=2019-06-16/

cat files from S3 using the hdfs API

The native aws s3 CLI does not allow to cat files. This can be overcome by using the following command which leverages the hdfs API. The command works as: copies the file to HDFS → runs cat command over the file on HDFS

hdfs dfs -cat <s3a://location>
